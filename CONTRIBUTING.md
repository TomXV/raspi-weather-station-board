# Contributing

Thanks for your interest!

## Development

```bash
bun install
bun run watch
```

## Pull Request Guide

1. Create a feature branch
2. Keep commits small and clear
3. Run checks before PR
   - `bun run build`
   - `bun run test`
4. Open a PR with:
   - what changed
   - why
   - screenshots (if UI changed)

## Style

- Keep UI readable on kiosk displays
- Avoid adding heavy dependencies unless necessary
- Prefer simple, maintainable code

## Bug Reports

Please include:
- device / screen size
- reproduction steps
- expected vs actual behavior
- screenshot or short clip
